# Pay Screen Keypad Redesign

## Goal
Convert the Pay screen to use a numeric keypad UI (like the HTML reference) and separate the NFC tap functionality into a new screen.

## User Requirements
1. **Confirm action**: Navigate to NFC screen with auto-activated NFC (no button press needed)
2. **Currency format**: Keep USDC format with BalanceCard font style (italic, large)
3. **Back button**: Add back button to header

## Implementation

### 1. Create `app/pay-nfc.tsx` (New File)

Contains all NFC logic moved from current `pay.tsx`:
- Auto-activates NFC on mount
- Shows "READY TO TAP" state with NfcRipple animation
- Handles processing/success/error states
- Receives amount via router params
- Has back button and home button
- Footer: "Secured by Bump Protocol v2.4"

### 2. Modify `app/pay.tsx`

Convert to keypad entry screen:
- Remove TextInput, add numeric keypad (3x4 grid: 1-9, ., 0, backspace)
- Amount display (read-only, large font, italic style matching BalanceCard)
- Format: `{amount} USDC`
- Back button in header
- Confirm button navigates to `/pay-nfc?amount={amount}`

### Key Design Elements
- Background: `COLORS.green400` (#4ade80)
- Amount display: fontSize 48, fontWeight 900, italic
- Keypad buttons: White background with neo-brutalism shadow
- Confirm button: Black background, full width, large text
- Back button: Arrow in bordered box
